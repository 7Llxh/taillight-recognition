## 1. 模块定位

特征提取网络是车型判别的**算法核心**。系统维护两个嵌入网络（同架构、分别训练）：

```
尾灯裁剪图(224) ─▶ [尾灯 ResNet-50 嵌入网络] ─▶ 512维 ─▶ L2归一化 ─▶ 尾灯特征库(主)
整车/车尾图(224) ─▶ [整车 ResNet-50 嵌入网络] ─▶ 512维 ─▶ L2归一化 ─▶ 整车特征库(兜底)
```

> **为何分别训练两个网络**：尾灯（小局部图）与整车（全局图）输入域差异大，特征空间不同，共享 backbone 效果不佳。两者同用 ResNet-50 架构与训练流程，但权重独立。

---

## 2. 需求与约束（决策）

| 项 | 决策 |
|---|---|
| 骨干网络 | **ResNet-50**（ImageNet 预训练，去分类头） |
| 度量损失 | **ArcFace 代理式**（margin=0.5, scale=64） |
| 特征维度 | **512 维** |
| 输入尺寸 | 224×224 |
| 每车型样本 | **20–30 张**（含亮灯+灭灯） |
| 库组织 | 统一库 + lit_status 状态过滤 |
| 检索度量 | cosine 相似度（L2 归一化后内积） |
| 训练框架 | PyTorch + pytorch-metric-learning（ArcFaceLoss） |

---

## 3. 网络结构

```
输入 224×224×3
  │
  ▼
ResNet-50 backbone（去 avgpool / fc）
  │
  ▼
AdaptiveAvgPool2d(1) → Flatten
  │
  ▼
Linear(2048, 512) → BatchNorm1d(512)        ← 嵌入头
  │
  ├── 训练分支 ──▶ ArcMarginProduct(512→num_classes, s=64, m=0.5) → CrossEntropyLoss
  │
  └── 推理分支 ──▶ L2 normalize → 512 维向量（入库/检索用）
```

- **嵌入头**：`Linear(2048→512) + BatchNorm`，BN 提升训练稳定性与特征分布一致性。
- **ArcFace head**：仅在训练时挂载，推理时不参与，输出 512 维归一化向量。
- **参数量**：~24M（ResNet-50），CPU 单图推理 ~30ms。

---

## 4. ArcFace 损失

### 4.1 原理
ArcFace 在嵌入向量与各类别权重间施加**角度 margin**，使同类收紧、异类推远，比纯 Triplet 训练更稳、无需难样本挖掘：

```
logit_i = s · cos(θ_i + m)   (对目标类)
logit_j = s · cos(θ_j)        (对非目标类)
loss    = CrossEntropy(softmax(logits), label)
```

### 4.2 超参

| 超参 | 值 | 说明 |
|---|---|---|
| margin m | 0.5（弧度≈28.6°） | 角度间隔 |
| scale s | 64 | logits 缩放，影响类间紧致度 |
| num_classes | 车型系列数 | 随库扩展增长 |

### 4.3 选型理由（vs Triplet）
- 小数据（20–30 张/车型）下训练更稳、收敛更快。
- 无需构造三元组与难样本挖掘，pipeline 简单。
- 类别代理天然适合"车型系列"这种离散类别结构。

## 5. 训练方案

| 项 | 配置 |
|---|---|
| 数据增强 | albumentations：HorizontalFlip、ColorJitter、RandomBrightnessContrast、RandomResizedCrop |
| 采样 | 普通随机 batch（ArcFace 对采样不敏感）；建议 batch 含多类多样本 |
| 优化器 | AdamW, lr=1e-4, weight_decay=1e-4 |
| 学习率 | Cosine Annealing |
| Epochs | 50，早停（验证 Recall@1 不升则停） |
| Batch size | 64 |
| 预训练 | ImageNet 权重初始化 backbone |

两个网络（尾灯 / 整车）用**相同超参分别训练**，仅训练集与 num_classes 不同。

## 7. 推理与入库

### 7.1 推理流程
```
裁剪图 → resize 224 → ImageNet 归一化 → ResNet-50 嵌入头 → 512 维 → L2 归一化 → 向量
```

### 7.2 入库
- 向量写入 FAISS `IndexFlatIP`（内积；配 L2 归一化向量 = cosine）。
- 元数据（车型、朝向、亮灭、样本路径、尾灯框）写入 `taillight_meta.json` / `vehicle_meta.json`，向量 id 与元数据 id 对齐。

### 7.3 状态过滤
- 尾灯库检索时按查询图 `lit_status` 过滤候选集（亮灯查亮灯样本，灭灯查灭灯样本）；无匹配状态时回退全库。

---

## 8. 检索策略

```
查询尾灯图 → 提取512维向量 → L2归一化
  │
  ▼
FAISS IndexFlatIP 检索 Top-K（按 lit_status 过滤）
  │
  ▼
相似度归一化为置信度 → 返回 Top-K 车型 + 置信度
```

- 置信度低于阈值 → 回退整车库（见 F20 结果融合）。

---

## 9. 评估方案

| 指标 | 目标 |
|---|---|
| 尾灯库 Recall@1（车尾视图） | ≥ 0.70 |
| 尾灯库 Recall@5（车尾视图） | ≥ 0.85 |
| 整车库 Recall@5（正/侧视图） | ≥ 0.60（兜底，预期低于尾灯库） |
| 特征分布 | t-SNE 可视化同类聚集、异类分离 |

- 分别评估尾灯库与整车库。
- 亮灯/灭灯子集分别统计 Recall，验证统一库 + 状态过滤的有效性。

## 11. 落地步骤

1. **数据准备**：标注工具产出尾灯/整车训练集（按车型+朝向组织）。
2. **实现 embedder**：ResNet-50 + 嵌入头 + ArcFace head。
3. **训练尾灯网络**：先小规模 10–20 车型跑通，再扩量。
4. **训练整车网络**：同流程。
5. **构建 FAISS 库**：入库 + 元数据 + 状态过滤。
6. **评估**：Recall@K + t-SNE，迭代超参。

## 12. 风险与对策

| 风险 | 对策 |
|---|---|
| 数据不足致 ArcFace 不收敛 | 先 10–20 车型验证流程；数据增强；ImageNet 预训练 |
| 相近车型尾灯混淆（如同品牌多车型） | 增加每车型样本；整车特征兜底辅助 |
| 灭灯与亮灯特征域差异大 | 统一库训练让模型学跨状态不变性；状态过滤减少跨域干扰 |
| 类别随库增长、需重训 | 库扩展时新车型加入重新训练；或采用增量策略（后续优化） |
