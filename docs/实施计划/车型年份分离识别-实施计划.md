# 车型年份分离识别 - 实施计划

> **实施状态（2026-07-20）**：阶段 0-5 子集验证完成（50 系列 × 每系列 30 图）。
> - survey：1175 系列 -> 剔除 <20 图 480 -> 保留 695 系列（28.16 万图）
> - 训练 R@1：整车 0.14 / 尾灯 0.20（每类样本少致过拟合，全量待扩）
> - 端到端跑通：recognize 输出车型 + 年份分离
> - **待办**：扩全量（28 万图，需批推理提速）+ 精度调优
> - 阈值调整：`MIN_IMGS_PER_SERIES=20`（原计划 10）

## 1. 背景与目标

当前系统（`data/raw`，5 车型）把"品牌+车型+年份"混成一个标签直接检索。引入 VMMRdb（9170 个 `make_model_year` 目录）后，需支持：

- **车型（make_model）识别**：可靠判别（尾灯/整车特征擅长）
- **年份识别**：模糊输出（年代范围 + 众数），分散时标"年份不确定"

尾灯特征对同年份区分力弱（同代尾灯几乎一样），年份本就难精确，故模糊输出，契合"年份不一定要准确"。

## 2. VMMRdb 数据画像

| 指标 | 数值 |
|---|---|
| `make_model_year` 目录 | 9,170 |
| 归并为车型系列（make_model） | 1,176 |
| 图片总数 | 285,086（约 28.5 万） |
| 每系列图片数 | 中位 33，范围 0~6895，均值 242 |

- `1900` 为缺年份占位符（如 `acura_integra_1900`）
- 样例：`acura_cl` 6年169图（1997~2003）、`acura_mdx` 16年974图、`acura_nsx` 仅1年1图（极稀疏）

## 3. 关键决策（已确认）

| 决策 | 选择 |
|---|---|
| 实施范围 | 先子集验证（≥30图系列，约600个）再扩全量 |
| 稀疏系列（<10图） | 剔除，识别到标"未知（样本不足）" |
| 年份输出 | 年代范围+众数（如"约1997-1999"），分散标"年份不确定" |

## 4. 架构调整

**整车库为主力，尾灯库为增强**（相对原系统"尾灯主+整车兜底"的反转）：

- **整车库**：全朝向（front/side/rear）整车裁剪入库。不依赖尾灯定位，覆盖广，车型识别主力。
- **尾灯库**：rear 图能定位尾灯的裁剪入库。rear 高精度增强（受部件检测器漏检限制）。
- **meta 增加 year 字段**；FAISS 检索自动带年份返回。
- **朝向分类器、部件检测器复用现有**（不重训，通用）。

**理由**：部件检测器仅 acura/audi 标注训练，VMMRdb 1176 车型尾灯漏检会放大（README 已记 buick/cadillac 漏检）。整车特征不依赖尾灯定位，作主力更稳。

---

## 5. 识别流程

```
输入图 -> 车辆检测 -> 朝向识别
  ├─ rear + 能定位尾灯 -> 尾灯库检索（高精度）
  └─ 其他 -> 整车库检索（主力）
-> matches（含 model_series + year + similarity）
-> 车型 = 候选 model_series 众数, 置信度 = 该车型 top1 相似度
-> 年份 = 该车型候选 year 统计:
     候选≥3 且 跨度≤2 -> "约{min}-{max}"
     否则            -> "年份不确定(候选 {min}-{max})"
-> 输出 {make_model, year_range, year_conf, confidence, topk} + 可视化
```

## 6. 实施阶段

### 阶段 0：数据准备
- 新增 `config.py`：`RAW_DIR`、`MIN_IMGS_PER_SERIES`（=20）、`SUBSET_MIN_IMGS`（=30）、设备等
- 新增 `survey_vmmr.py`：扫 VMMRdb，归并 make_model，生成系列清单 `data/vmmr_series.json`（`make_model -> {years, img_count, img_paths}`），剔除 <10 图系列，标记子集
- `data/vmmr_series.json` 入 `.gitignore`（数据清单大）

### 阶段 1：数据集生成（批处理 + 断点续传）
改 `make_vehicle_dataset.py` / `make_taillight_dataset.py`：
- 数据源：`data/raw` → `RAW_DIR`（survey 产 `vmmr_series.json`）
- 归并：目录=make_model，文件名带 year：`{make_model}__{year}__{idx:04d}.jpg`
- `make_vehicle`：全朝向整车裁剪入库（主力）
- `make_taillight`：rear + 能定位尾灯的裁剪
- 加 tqdm 进度 + 断点续传（已处理清单跳过）+ YOLO 批推理提速
- `--subset` 仅处理 ≥30 图系列（验证）；全量模式处理所有 ≥10 图系列

### 阶段 2：训练
- `train_embedder.py vehicle`：整车 embedder，类别=make_model（子集约 600 类），架构不变
- `train_embedder.py taillight`：尾灯 embedder（若样本足）
- 超参沿用，看效果再调（batch/epochs）

### 阶段 3：建库（加 year）
改 `build_library.py`：
- meta entry 加 `year`：从文件名 `{make_model}__{year}__{idx}` 解析；`1900`/缺年份标 `None`
- 其余不变

### 阶段 4：识别流程
改 `recognize.py`：
- 新增 `_judge_model(matches)`：车型=候选 model_series 众数，置信度=该车型 top1 相似度
- 新增 `_judge_year(matches, model)`：该车型候选 year 统计，输出范围/众数/不确定
- 输出 JSON 加 `year_range`、`year_conf`
- 可视化：框标"车型 年份范围 置信度"
- 双路径保留（rear+尾灯→尾灯库；其他→整车库主力）

### 阶段 5：验证与扩展
- `check_taillight_detection.py` 在 VMMRdb 抽样：量化部件检测器漏检，决定尾灯库可行性
- 子集测试：跑 VMMRdb 测试图，看车型准确率 + 年份合理性
- 验证 OK → 关子集模式，扩全量（重跑阶段 1-3，断点续传）

## 7. 关键实现点

- **make_model 归并 + year 解析**：`d.rsplit('_', 1)` → `(make_model, year)`。`1900` 当缺年份（`None`）。
- **配置化**：`RAW_DIR` 等入 `config.py`，避免硬编码。VMMRdb 已复制入 `data/raw`（统一数据源）。
- **批处理/断点续传**：make_* 加已处理清单（跳过）+ tqdm + YOLO 批推理。28.5 万图必备。
- **文件名带 year**：`{make_model}__{year}__{idx}.jpg`，build_library 解析。
- **年份统计**：候选 year 集中给范围，分散标不确定。`1900`/`None` 不参与年份统计。
- **faiss_index.search 不改**：返回 `{**meta, similarity}`，meta 加 year 自动带。

## 8. 风险与应对

| 风险 | 应对 |
|---|---|
| 部件检测器 VMMRdb 漏检 → 尾灯库覆盖不足 | 整车库主力兜底；阶段 5 check 量化 |
| 全量耗时 ~20h（数据集12h+训练6h+建库2h） | 子集验证先行；断点续传；批推理 |
| 年份区分力弱 | 模糊输出（范围+众数）设计已容纳 |
| 1176 类 ArcFace 训练显存/收敛 | 子集先验；超参可调 |
