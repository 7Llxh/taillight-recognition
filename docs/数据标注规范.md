# 数据标注规范

| 项 | 内容 |
|---|---|
| 标注工具 | [annotate_tool.py](../annotate_tool.py) |
| 产出 | `data/annotations/{车型}_{图名}.json` |
| 对应需求 | [需求分析.md](需求分析.md) §6、F1 |

---

## 1. 标注内容

每张图标注：
- **车型**：`data/raw/{车型}/` 目录名自动预填
- **朝向**：front / rear / side（人工选）
- **部件框**：人工画（8 类）
- **车辆框**：自动检测（可校正）

亮/灭灯状态自动判定（`_judge_lit`），不需人工标。

---

## 2. 部件类别（8 类）

| index | 类别 | 说明 |
|---|---|---|
| 0 | taillight | 尾灯（**核心**） |
| 1 | headlight | 前灯 |
| 2 | mirror | 后视镜 |
| 3 | window | 车窗 |
| 4 | wheel | 轮胎 |
| 5 | plate | 车牌 |
| 6 | grille | 格栅 |
| 7 | exhaust | 排气管 |

> 类别顺序与 `make_parts_dataset.py` 的 PARTS 列表一致，index 用于 YOLO 标签。

---

## 3. 朝向

- **front**：正面（车头）
- **rear**：反面（车尾）
- **side**：侧面

首期不区分左右侧与斜角。

---

## 4. 标注 JSON 格式

```json
{
  "image": "data/raw/acura_mdx_2007/xxx.jpg",
  "model": "acura_mdx_2007",
  "view": "rear",
  "vehicle_box": [x1, y1, x2, y2],
  "parts": [
    {"label": "taillight", "box": [x1, y1, x2, y2]},
    {"label": "wheel", "box": [x1, y1, x2, y2]}
  ]
}
```

---

## 5. 数据组织

```
data/
├── raw/{车型}/           # 原始图（车尾/正面/侧面，亮灯+灭灯）
├── annotations/          # 标注 JSON
├── orientation_dataset/  # 朝向数据集（自动从 annotations 生成）
├── parts_dataset/        # 部件数据集（自动）
├── taillight_dataset/    # 尾灯训练集（自动，部件检测器裁）
├── vehicle_dataset/      # 整车训练集（自动）
└── features/             # FAISS 库
```

---

## 6. 数据划分

| 数据集 | 划分 | 说明 |
|---|---|---|
| 朝向 / 部件 | 8:2（train:val） | 从 annotations 生成 |
| 尾灯 / 整车特征 | 8:1:1（train:val:test） | 按车型分层，test 不建库不训练 |

---

## 7. 车型命名

`data/raw/{品牌}_{车型}_{年款}/`，如：
- `acura_mdx_2007`
- `audi_a4_2011`
- `bmw_335i_2015`

车型名即特征网络的类别标签（目录名）。

---

## 8. 何时需要标注

| 场景 | 是否标注 |
|---|---|
| 训练部件检测器/朝向分类器（首次） | **需要**（已用 acura+audi 标注训好） |
| 加新车型 | **先 check**：`python check_taillight_detection.py {车型}` 产漏检队列；平均尾灯/rear图 ≈2 不标注，<1.5 才补标注 |
| 提升部件检测器在新车型精度 | 可选补标注 + 重训 |

> 部件检测器/朝向分类器不分车型（学部件/朝向外观，通用），加新车型一般不用标注；只有漏检才补。

---

## 9. 标注操作（annotate_tool.py）

```bash
python annotate_tool.py
```

- 左键拖动画部件框（选当前类别）
- 朝向 radio 选 front/rear/side
- `a`/`d` 上一张/下一张，`s` 保存，"跳到未标注"跳过已标，`m` 跳下一车型未标注图（每车型标几张就换）
- 自动检测车辆框（绿框参考，可校正）

**多车型轮流标注**：扩标注时每车型标几张尾灯框，按 `m` 换下一车型，覆盖多品牌尾灯形状，不必标完整个车型。

**标注增强（漏检补标）**：check 产出 `data/annotate_queue.json` 后，在工具内按 `n` 跳到下一张需增强图（复用 check 已算的车辆框，跳过重检测），画尾灯框 + `s` 保存自动出队，队列递减直至空。
