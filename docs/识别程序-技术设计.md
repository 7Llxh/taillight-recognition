# 识别程序技术设计（F11-F21）

| 项 | 内容 |
|---|---|
| 所属子系统 | 识别程序（Subsystem B） |
| 对应需求 | [需求分析.md](需求分析.md) F11-F21 |
| 输入 | 单张图片（支持多车） |
| 输出 | 车型 + 置信度 + Top-K + 可视化 |
| 实现 | [recognize.py](../recognize.py) |

---

## 1. 模块定位

识别程序是端到端判别入口：输入图像 -> 检测/朝向/尾灯定位/特征检索 -> 判别车型 + 可视化。按朝向双路径分流（尾灯主 + 整车兜底）。

---

## 2. 总体流程

```
图像 -> 车辆检测(YOLOv8s) -> 对每辆车:
  朝向识别(YOLOv8n-cls)
  ├─ rear   -> 部件检测器尾灯 -> 尾灯特征(ResNet50) -> 检索尾灯库(主) -> top1
  └─ front/side -> 整车特征 -> 检索整车库(兜底) -> top1
-> 可视化(框 + 车型 + 置信度) + JSON
```

---

## 3. 朝向分流

| 朝向 | 路径 | 检索库 |
|---|---|---|
| rear（车尾） | 尾灯主路径（精度高） | 尾灯库 |
| front / side / uncertain | 整车兜底 | 整车库 |

---

## 4. 尾灯主路径（rear）

1. `detect_taillights` 定位尾灯框（部件检测器）
2. 取置信度最高尾灯 -> `_judge_lit` 判亮灭
3. `crop_and_normalize` 224 -> `extract_features`（尾灯网络）512 维
4. `search` 尾灯库（`lit_status_filter` 按亮灭过滤；无同状态回退全库）
5. top1 车型 + 置信度（cosine 相似度）

---

## 5. 整车兜底（front/side）

1. 整车裁剪（detect_vehicle_full 的 box）-> `crop_and_normalize` 224
2. `extract_features`（整车网络）512 维
3. `search` 整车库
4. top1 车型 + 置信度

---

## 6. 置信度与回退

- 置信度 = 检索相似度（cosine，0-1）
- `MIN_CONF=0.5`：低于此标"未知(置信度低)"（后续可回退整车库）
- 尾灯未检测到 -> 标"未检测到尾灯"
- 整车兜底库未构建 -> 标"兜底库未构建"

---

## 7. 可视化输出

- **结果图**：车辆框 + 车型 + 置信度 + 朝向
  - 红框 = 亮灯尾灯、绿框 = 灭灯尾灯、黄框 = 兜底/未检测
- **JSON**：`{box, view, lit, top1, confidence, topk}`

输出到 `{图名}_recognize/`：`{图名}_recognize.jpg` + `{图名}_result.json`。

---

## 9. 指标（端到端验证）

| 测试 | 朝向 | 路径 | top1 | 置信度 |
|---|---|---|---|---|
| acura 车尾 | rear | 尾灯主 | acura_mdx_2007 | 0.997 |
| audi 车尾 | rear | 尾灯主 | audi_a4_2011 | 0.994 |
| bmw 正面 | front | 整车兜底 | bmw_335i_2015 | 0.972 |

test 集尾灯检索 Recall@1=90%（acura 100%、audi 75%、bmw 100%）。

## 10. 依赖

- 车辆检测：detect_vehicle_full
- 朝向：[orientation.py](../orientation.py) `detect_orientation`
- 尾灯定位：[detect_vehicle.py](../detect_vehicle.py) `detect_taillights`、`_judge_lit`、`crop_and_normalize`
- 特征：[embedder.py](../embedder.py) `extract_features`（尾灯/整车网络）
- 检索：[faiss_index.py](../faiss_index.py) `search`
- 库：`data/features/{taillight,vehicle}_index.faiss` + meta

## 11. 落地步骤
