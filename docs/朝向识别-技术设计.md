# 朝向识别模块技术设计（F3）

| 项 | 内容 |
|---|---|
| 所属子系统 | 建库程序 / 识别程序（公共前置模块） |
| 对应需求 | [需求分析.md](需求分析.md) F3 |
| 朝向粒度 | 正面（车头）/ 反面（车尾）/ 侧面，共 3 类 |
| 骨干 | **YOLOv8n-cls**（ultralytics 分类模式） |
| 依赖 | ultralytics（已装）、标注数据（朝向标签） |

---

## 1. 模块定位

朝向识别是整个识别流程的**分流枢纽**：

```
车辆检测 ──▶ 【朝向识别】 ──┬─▶ 反面(车尾) ──▶ 尾灯定位 ──▶ 尾灯特征库检索（主路径）
                            ├─▶ 正面(车头) ─▶ 整车特征库检索（兜底）
                            ├─▶ 侧面      ──▶ 整车特征库检索（兜底）
                            └─▶ 不确定    ──▶ 整车特征库检索（兜底）
```

朝向判断的准确性直接决定下游走尾灯库还是整车库。其中**正面/反面的区分尤为关键**--判错会把本该走尾灯库的车尾图送进整车库，丢失高精度判别能力。

---

## 2. 需求与约束

- **类别**：`front`（正面/车头）、`rear`（反面/车尾）、`side`（侧面），共 3 类。
- **首期不区分左右侧**：侧面合并为一类；水平翻转数据增强有效（因左右侧对称）。
- **输入**：YOLOv8 检测出的车辆裁剪图（来自 [detect_vehicle_full](detect_vehicle.py)）。
- **输出**：朝向类别 + 置信度，支持"不确定"状态以触发兜底。
- **运行环境**：Windows 11 本地 CPU，单图推理需轻量。
- **准确性目标**：整体准确率 ≥ 90%，反面（车尾）召回率 ≥ 90%（避免漏判车尾）。

---

## 3. 输入输出规格

**输入**

```python
vehicle_crop: np.ndarray   # BGR, H×W×3, 车辆裁剪图（非定尺寸）
```

**输出**

```python
{
    "view": "rear",          # front | rear | side | uncertain
    "confidence": 0.93,      # 分类置信度（uncertain 时为 0）
    "raw_probs": {           # 原始三类概率
        "front": 0.02, "rear": 0.93, "side": 0.05
    }
}
```

---

## 4. 方案选型

| 方案 | 数据需求 | 训练成本 | 准确率预期 | 生态统一 | 评价 |
|---|---|---|---|---|---|
| **YOLOv8-cls** | 中（3 类各 ~50） | 低（ultralytics 开箱即用） | 高（≥90%） | ✅ 统一 ultralytics | **★ 推荐** |
| MobileNetV3 | 中 | 中（自写训练循环） | 高 | ❌ 多 timm 栈 | 更轻但开发成本高 |
| 纯规则（尾灯有无） | 无 | 无 | 低（HSV 不准） | - | 已验证不可靠，弃用 |
| CLIP 零样本 | 无 | 无 | 中（细粒度不稳） | ❌ | 依赖大模型 |

### 推荐：YOLOv8n-cls

- **统一生态**：车辆检测（yolov8s）、朝向分类（yolov8n-cls）、部件检测（yolov8）都用 ultralytics 一套库、一套 API、一套数据格式（ImageFolder）。
- **开发成本低**：训练分类只要数据按 ImageFolder 放好，一行 `model.train()` 即训。
- **精度足够**：3 分类是粗粒度，YOLOv8n-cls 轻松胜任。
- **无尾灯先验**：之前设计有"成对尾灯覆写 rear"的先验，但 HSV 尾灯检测在实测中不准（已从标注工具移除），故去掉先验，纯靠分类器学车辆整体外观特征（灭灯也鲁棒）。

---

## 5. 推荐方案总体设计

```
车辆裁剪图
    │
    ▼
[YOLOv8n-cls 分类] ──▶ 三类概率 (front/rear/side)
    │
    ▼
[置信度判定]
    │   - max_prob ≥ 0.6 -> 输出对应朝向
    │   - 否则 -> uncertain（触发整车兜底）
    ▼
输出 view + confidence
```

## 6. 数据准备

### 6.1 来源
- 标注工具产出的车辆裁剪图 + 朝向标签（来自 `data/annotations/*.json` 的 `vehicle_box` + `view`）。
- 从标注 JSON 提取车辆裁剪图，按 `view` 分到 ImageFolder。

### 6.2 数据格式（ImageFolder）
```
data/orientation_dataset/
├── train/
│   ├── front/   *.jpg
│   ├── rear/    *.jpg
│   └── side/    *.jpg
├── val/
│   ├── front/ rear/ side/
└── test/
    ├── front/ rear/ side/
```

### 6.3 规模与划分
- 初版每类 30–50 张（冷启动），半自动回灌扩充。
- 训练 : 验证 : 测试 = 8 : 1 : 1，按车型分层。
- 侧面样本若不足，水平翻转扩充（首期不区分左右，翻转有效）。

---

## 7. 模型设计

- **骨干**：YOLOv8n-cls（ImageNet 预训练，ultralytics 首次使用自动下载 `yolov8n-cls.pt`）。
- **输入**：224×224×3，RGB。
- **输出**：3 类 softmax 概率。
- **参数量**：~3M，CPU 单图 <20ms。

---

## 8. 训练方案

ultralytics 训练开箱即用：

```python
from ultralytics import YOLO
model = YOLO("yolov8n-cls.pt")           # ImageNet 预训练
model.train(data="data/orientation_dataset", epochs=50, imgsz=224, batch=64)
```

| 项 | 配置 |
|---|---|
| 优化器 | AdamW（ultralytics 默认） |
| 学习率调度 | Cosine（默认） |
| Epochs | 50，早停 |
| 数据增强 | ultralytics 自带（翻转/颜色/裁剪） |
| 类别不均衡 | 侧面样本少时加权或重采样 |

## 9. 推理与后处理

```python
def detect_orientation(vehicle_crop, conf_threshold=0.6):
    model = YOLO("runs/cls/train/weights/best.pt")  # 训练后的朝向分类器
    res = model.predict(vehicle_crop, imgsz=224)[0]
    probs = res.probs
    top_idx = int(probs.top1)
    view = ["front","rear","side"][top_idx]   # 按训练类别顺序
    conf = float(probs.top1conf)
    if conf < conf_threshold:
        view = "uncertain"
    return {"view": view, "confidence": conf, ...}
```

### 置信度阈值
- `conf_threshold = 0.6`：低于此值输出 `uncertain`，识别程序走整车兜底，避免错误分流。

### 边界场景
| 场景 | 处理 |
|---|---|
| 后侧方（一个尾灯可见）| 学整体外观判断；若偏后则归 rear（能走尾灯判别） |
| 灭灯尾灯 | 分类器学整体特征，不依赖尾灯颜色，灭灯也鲁棒 |
| 斜 45° 视角 | 首期归入最近的正/侧类；占比高则后续扩"斜角"类 |

---

## 10. 评估方案

| 指标 | 目标 |
|---|---|
| 整体准确率 | ≥ 90% |
| 反面（车尾）召回率 | ≥ 90%（关键，避免漏判走尾灯库） |
| 正/反混淆率 | 越低越好（重点观测混淆矩阵该格） |
| 推理延迟（CPU） | < 20ms / 辆 |

输出**混淆矩阵**，重点关注 `front↔rear` 格。

## 12. 落地步骤

1. **标注朝向**：用标注工具标每类 30–50 张（朝向 radio 选 front/rear/side）。
2. **生成数据集**：从 `data/annotations/*.json` 提取车辆裁剪 + 朝向，组织成 ImageFolder。
3. **训练初版**：`YOLO("yolov8n-cls.pt").train(data=...)`。
4. **评估**：测试集计算准确率 + 混淆矩阵。
5. **半自动回灌**：用初版预标剩余图片朝向，人工校正错误，扩充训练集重训。
6. **接入主流程**：在 `detect_tail_light` 插入分流调用。

---

## 13. 风险与对策

| 风险 | 对策 |
|---|---|
| 正/反混淆导致车尾漏判走兜底 | 关注反面召回率指标；数据多含正后视图 |
| 侧面样本不足致分类偏差 | 水平翻转增强 + 类别加权 |
| 斜角视角归错类 | 首期容忍；占比高则扩"斜角"类 |
| 初版模型准确率不足影响半自动效率 | 初版可接受较低准确率，人工校正成本低；迭代提升 |
