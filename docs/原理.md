# 系统原理

汽车尾灯车型识别系统：通过**图像检索**判别车型。构建尾灯/整车特征向量库，输入图像提取特征后检索库，最相似的车型即判别结果。

## 总览

```
输入图 -> 车辆检测 -> 朝向识别
   ├─ 车尾 rear   -> 尾灯定位 -> 尾灯特征 -> 检索尾灯库（主，精度高）
   └─ 正面/侧面  -> 整车特征 -> 检索整车库（兜底）
-> 车型 + 置信度 + Top-K + 可视化
```

**核心思路**：不直接分类车型，而是把图像编码成特征向量，在库里找最相似的。度量学习（ArcFace）让同车型向量靠近、异车型远离，检索即可判别。这种方式可扩展——新车型只需入库，不用重训分类头。

---

## 1. 车辆检测：YOLOv8s

**为什么**：输入图可能含多车/背景，需先框出每辆车，逐辆判别。

**怎么做**：YOLOv8s（COCO 预训练）检测 car/bus/truck，后处理过滤面积过小（<0.5% 图面积）或宽高比异常（<0.25 或 >4）的框，去除招牌/栏杆等误检。

**原理**：卷积神经网络多层提取特征（边缘 → 形状 → 部件），输出"该位置有车辆"的概率和边界框。

---

## 2. 朝向识别：YOLOv8n-cls

**为什么**：车尾有尾灯（可走高精度尾灯库），正面/侧面没尾灯（走整车兜底）。朝向决定走哪条路，是分流枢纽。

**怎么做**：YOLOv8n-cls 三分类（front/rear/side），学整车整体外观。置信度 < 0.6 输出 uncertain（走兜底，避免错误分流）。

**为什么不用 HSV 尾灯判朝向**：HSV 尾灯检测在灭灯/红色干扰下不准，已弃用。改纯分类器学整体外观（不依赖尾灯颜色，灭灯也鲁棒）。

---

## 3. 尾灯定位：部件检测器

**为什么**：车尾判别靠尾灯特征，需先在车内定位尾灯框。

**怎么做**：部件检测器（YOLOv8s，9 类部件）在车辆裁剪图上检测，筛 taillight 类。taillight mAP50=0.99、recall=0.967。

**为什么不用 HSV 红色分割**（旧方案，已弃用）：
- 灭灯尾灯不发光、红色不明显 → HSV 漏检
- 风景红色物体（招牌/衣物/建筑）→ 误检
- 左右对称配对在侧视/单灯可见时 → 失败

部件检测器用深度学习直接识别尾灯（**不依赖颜色**），解决了上述问题，灭灯也鲁棒。

**风险**：部件检测器只训练了 acura+audi 标注，在其他车型可能漏检（如 buick/cadillac，58/62、43/45 张 rear 图无尾灯）。用 `check_taillight_detection.py` 检查，漏检则补标注 + 重训部件。

---

## 4. 亮/灭灯判定

**为什么**：亮灯/灭灯尾灯特征域不同，检索时按状态过滤（亮灯查亮灯样本，灭灯查灭灯），减少跨域干扰。

**怎么做**：尾灯裁剪图转 HSV，取 V 通道（亮度）均值，≥ 150（TAILLIGHT_MIN_V）为亮灯。不建 HSV mask，只取亮度通道。

> 阈值 150 为经验值，夜间场景为主；白天灭灯尾灯 V 可能偏高。

---

## 5. 特征提取：ResNet-50 + ArcFace

**为什么**：把尾灯/整车图像编码成 512 维向量，供检索。度量学习让同车型向量靠近、异车型远离。

**怎么做**：

```
输入 224×224×3
  -> ResNet-50 backbone（去 avgpool/fc）
  -> AdaptiveAvgPool2d -> Flatten
  -> Linear(2048→512) + BatchNorm1d   ← 嵌入头
  -> L2 normalize -> 512 维向量（入库/检索用）
```

训练时挂 ArcFace head（margin=0.5, scale=64）+ CrossEntropyLoss；推理时只输出 512 维归一化向量。

**为什么 ArcFace 而非三元组损失**：
- 小数据（20-30 张/车型）下收敛更稳
- 无需构造三元组/难样本挖掘，pipeline 简单
- 类别代理天然适合"车型系列"这种离散类别结构

**两个网络**：尾灯网络（尾灯裁剪输入）+ 整车网络（整车图输入），同架构分别训练（输入域差异大，不共享权重）。

---

## 6. 检索：FAISS

**为什么**：从特征库找最相似的车型，比线性比较快（万级向量 <100ms）。

**怎么做**：
- **索引**：IndexFlatIP（内积；配 L2 归一化向量 = cosine 相似度）
- **检索**：查询向量 → Top-K 候选
- **状态过滤**：尾灯库按 lit_status 过滤（亮灯查亮灯，灭灯查灭灯；无同状态回退全库）
- **置信度** = 相似度（0-1）

**库组织**：统一库（亮/灭灯样本共库）+ 检索时按状态过滤，而非分亮灭子库。共库训练让模型学跨状态不变性，过滤减少跨域干扰。

---

## 7. 双路径融合

**为什么**：车尾有尾灯（精度高），正面/侧面没尾灯（兜底）。

**怎么做**：
- rear → 尾灯主路径（尾灯库检索，精度高）
- front/side/uncertain → 整车兜底（整车库检索）
- 尾灯置信度过低（尾灯被遮挡/灭灯且库内无灭灯样本）→ 回退整车兜底

---

## 8. 输出

- 车型 + 置信度 + Top-K 候选
- **可视化**：结果图（车辆框 + 车型 + 置信度；红=亮灯尾灯、绿=灭灯、黄=兜底）+ JSON（结构化结果）

---

## 关键设计决策汇总

| 决策 | 选择 | 原因 |
|---|---|---|
| 判别方式 | 图像检索（非分类） | 可扩展，新车型入库即可，不用重训分类头 |
| 尾灯定位 | 部件检测器（非 HSV） | 灭灯鲁棒、无红色干扰 |
| 朝向判别 | YOLOv8n-cls（非 HSV 尾灯） | HSV 不准，纯外观分类稳 |
| 度量学习 | ArcFace（非三元组） | 小数据稳、无需难样本挖掘 |
| 双库 | 尾灯主 + 整车兜底 | 车尾高精度 + 正侧兜底 |
| 亮灭库 | 统一库 + 状态过滤 | 共库训练跨状态不变性，检索过滤减干扰 |

## 数据流

```
原始图 data/raw/{车型}/
  -> 标注 annotate_tool -> data/annotations/
  -> 训练集 make_*_dataset -> data/*_dataset/
  -> 训练 train_* -> runs/
  -> 建库 build_library -> data/features/
  -> 识别 recognize -> {图名}_recognize/
```
